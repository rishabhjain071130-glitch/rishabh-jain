# 02 — BRAND IDENTITY

Version: 1.0

---

# Brand Purpose

This portfolio is not a resume.

It is a digital representation of how I think, learn, solve problems, and build technology.

Every page should increase trust.

Every interaction should communicate engineering quality.

Every animation should have purpose.

Every project should demonstrate problem-solving rather than simply showing code.

---

# Brand Positioning

Rishabh Jain is an engineering student focused on building secure and intelligent software.

The portfolio should communicate that he is not only learning technologies but also applying them to solve meaningful real-world problems.

This website should feel like the online presence of a future software engineer and technology founder.

---

# Brand Promise

Building secure, intelligent, and scalable software that creates real-world impact.

---

# Long-Term Direction

The long-term vision combines both engineering and entrepreneurship.

Primary Goal

• Become a highly skilled Software Engineer
• Build products that solve meaningful problems
• Start technology ventures in AI and Cyber Security
• Continue learning through real-world projects

The portfolio should grow with this journey instead of requiring redesigns every year.

# Typography Philosophy

Large headings.

Comfortable reading width.

Generous spacing.

Minimal font variety.

Typography should communicate engineering precision.

---

# Icon Style

Minimal.

Outline icons.

Rounded corners.

Consistent stroke width.

Avoid colorful illustration packs.

---

# Photography

Use a real professional portrait.

Never use AI-generated faces.

Never use stock photos.

Until a real image exists, use an elegant placeholder.

---

# Motion Philosophy

Motion should guide attention.

Never distract.

Animations should feel smooth and intentional.

Avoid exaggerated bouncing.

Avoid excessive blur.

Avoid gimmicks.

Preferred animation duration

150–300ms

Scrolling should feel continuous.

---

# Interaction Principles

Buttons should always provide feedback.

Hover states should be subtle.

Keyboard navigation must be fully supported.

Every interactive element must clearly communicate its state.

---

# Accessibility

Meet WCAG AA standards wherever possible.

High contrast.

Visible keyboard focus.

Semantic HTML.

Reduced motion support.

Screen reader compatibility.

---

# Content Principles

Never invent achievements.

Never fabricate statistics.

Never claim experience that doesn't exist.

Learning is valuable.

Honesty builds trust.

---

# Success Criteria

A recruiter should leave thinking

"This student has strong engineering fundamentals."

"This portfolio feels thoughtfully designed."

"I want to learn more."

"I would interview this person."

---

# Future Evolution

As experience grows

Projects will evolve.

Experience will expand.

Skills will improve.

The identity should remain consistent.

The portfolio should scale naturally without requiring a complete redesign.

---

# Final Rule

Every design decision should answer one question.

Does this increase trust?

If the answer is no,

do not build it.
