# 11 — RECRUITER EXPERIENCE PLAYBOOK

Version: 1.0

# Purpose

This document defines how recruiters, hiring managers, engineering leads, and startup founders should experience the portfolio.

The objective is not simply to present information.

The objective is to create confidence.

Every section should answer one question before the visitor asks it.

---

# Primary Audience

Software Recruiters

Engineering Managers

Technical Interviewers

Startup Founders

Hiring Committees

---

# Recruiter Psychology

A recruiter usually spends less than one minute deciding whether to continue reviewing a candidate.

The portfolio must therefore communicate credibility immediately.

Visitors should never have to search for important information.

Trust must be built progressively.

---

# First Five Seconds

The visitor must immediately understand:

• Who is Rishabh Jain?

• What does he build?

• What is his specialization?

• Is he worth exploring further?

If any of these answers require scrolling,

the Hero has failed.

---

# First Fifteen Seconds

The visitor should naturally continue scrolling.

They should immediately discover

Professional Identity

Current Focus

Projects

Engineering Mindset

Not a list of technologies.

---

# First Thirty Seconds

The recruiter should understand

This person builds projects.

This person learns continuously.

This person understands software engineering.

This person communicates clearly.

---

# First Minute

The visitor should already trust

The projects.

The architecture.

The engineering quality.

The presentation.

The attention to detail.

---

# Two Minutes

The recruiter should feel

"I want to interview this candidate."

The decision should come naturally.

Never force it.
